# X 中文转英文翻译插件 — 需求功能文档

## 1. 产品概述

在 X (Twitter) 的回复/发推编辑器中，为中文用户提供一键（或自动）翻译为英文并替换原文的功能，翻译结果可直接发送。

## 2. 核心功能

### 2.1 翻译按钮

- 在 X 的回复/发推编辑器工具栏中注入「中→EN」按钮
- 当编辑器内检测到中文时，按钮高亮显示（蓝色填充 + 白色文字）
- 点击按钮：调用配置的 AI API 将当前编辑器内的中文翻译为英文
- 翻译完成后自动替换编辑器内容，用户可直接点击 X 的发送按钮发布
- 按钮状态：
  - **默认**：透明背景 + 蓝色边框，显示「中→EN」
  - **检测到中文**：蓝色填充 + 白色文字（`has-chinese` 状态）
  - **翻译中**：半透明 + 脉冲动画，显示「翻译中...」
  - **未检测到中文**：普通状态，点击提示「未检测到中文内容」

### 2.2 自动翻译（可选）

- 用户可在设置面板中开启/关闭
- 开启后：停止输入 1.5 秒后自动触发翻译
- 避免重复翻译已翻译过的相同内容（`lastTranslatedText` 缓存）
- 正在翻译中不会触发新的翻译任务（`isTranslating` 锁）

### 2.3 多 AI 提供商支持

| 提供商 | 类型 | 默认模型 | Key 前缀 |
|--------|------|----------|----------|
| Claude (Anthropic) | Anthropic API | claude-sonnet-4-20250514 | `sk-ant-` |
| 硅基流动 (SiliconFlow) | OpenAI 兼容 | Qwen/Qwen2.5-72B-Instruct | 无 |
| DeepSeek | OpenAI 兼容 | deepseek-chat | `sk-` |
| 百炼 (阿里云) | OpenAI 兼容 | qwen-turbo | `sk-` |
| OpenAI 兼容 (自定义) | OpenAI 兼容 | gpt-4o | `sk-` |

- 用户可自定义 API 地址（仅 OpenAI 兼容模式）
- 切换提供商时自动填充对应的 API 地址、默认模型、Key 格式提示
- 各提供商可选模型列表不同，在提供商配置中定义

### 2.4 设置面板（Popup）

- **API Key 输入**：密码模式，支持显示/隐藏切换（👁 按钮）
- **API 地址**：非自定义模式只读显示，自定义模式可编辑
- **提供商选择**：下拉菜单切换，切换时自动更新 UI
- **自动翻译开关**：滑动开关控件
- **保存设置**：写入 `chrome.storage.sync`，保存后显示成功/失败提示
- **格式校验**：保存时检查 API Key 是否为空、格式是否符合当前提供商要求
- **使用方法指引**：4 步操作说明

## 3. 技术实现

### 3.1 编辑器注入

- **内容脚本**：`content.js` 通过 `MutationObserver` 监听 DOM 变化
- **目标元素**：`[data-testid="tweetTextarea_0"]` 或 `[contenteditable="true"][role="textbox"]`
- **按钮位置**：三级回退策略
  1. 优先插入工具栏（`[role="toolbar"]` 或 `[data-testid="toolBar"]`）
  2. 次选 compose 工具栏行（含多个 SVG 图标的 flex 行）
  3. 兜底方案：浮动定位到编辑器右侧
- **防重复绑定**：`WeakSet` 跟踪已绑定事件监听器的编辑器，避免内存泄漏和重复触发

### 3.2 文本替换

- **主流程**：`execCommand('selectAll')` → `execCommand('delete')` → 重建光标 → `execCommand('insertText')`
- **回退方案**：直接操作 DOM（`innerHTML` + `appendChild`）+ 模拟 `InputEvent` 事件链
- **翻译后处理**：光标自动移至文本末尾，确保可继续编辑
- **空编辑器保护**：`delete` 后若无子节点，自动创建空文本节点，避免 `setStart` 失败

### 3.3 翻译 API 调用

- **Prompt**：固定翻译提示词「请将以下中文翻译成自然流畅的英文。只输出翻译结果，不要任何解释或额外内容」
- **Anthropic API**：`x-api-key` + `anthropic-version: 2023-10-01` + `anthropic-dangerous-direct-browser-access: true`
- **OpenAI 兼容 API**：`Authorization: Bearer {key}` + 标准 Chat Completions 格式
- **错误处理**：
  - HTTP 状态码 + JSON 解析保护，非 JSON 响应降级为 HTTP 错误码
  - 空结果校验：翻译返回空字符串时提示用户重试
  - `try/catch` 保护 `response.json()`，防止 502/503 HTML 错误页导致二次崩溃

### 3.4 配置同步

- 内存缓存（`cachedConfig`）+ `chrome.storage.sync` 持久化
- 监听 `chrome.storage.onChanged` 实时同步设置变更
- 页面加载时自动从 storage 初始化配置
- 安全降级：`chrome.storage` 不可用时返回空对象，不阻塞运行

### 3.5 翻译按钮 DOM 恢复

翻译后 X 的 React 组件重渲染可能移除或清空按钮，通过以下方式恢复：

1. **`handleEditorInput`**：每次编辑器内容变化时检查按钮是否存在
2. **`doTranslate` finally 块**：翻译完成后立即检查
3. **延迟检查（500ms）**：处理 React 异步重渲染场景

## 4. 权限要求

| 权限 | 用途 |
|------|------|
| `activeTab` | 操作当前激活标签页 |
| `storage` | 保存/读取用户配置 |
| `host_permissions: x.com/*` | 内容脚本注入 |
| `host_permissions: twitter.com/*` | 内容脚本注入（旧域名兼容） |
| `host_permissions: api.anthropic.com/*` | Claude API 调用 |
| `host_permissions: api.siliconflow.cn/*` | 硅基流动 API 调用 |
| `host_permissions: api.deepseek.com/*` | DeepSeek API 调用 |
| `host_permissions: dashscope.aliyuncs.com/*` | 百炼 API 调用 |
| `host_permissions: https://*/*` | 自定义 API 地址（兜底授权） |

## 5. 交互流程

```
用户在 X 点击回复/发推
  ↓
MutationObserver 检测到新编辑器 DOM 节点
  ↓
注入翻译按钮 + 绑定 input/focus 事件监听
  ↓
用户输入中文内容 → 按钮高亮（蓝色填充）
  ↓
方式一：点击翻译按钮
方式二：自动翻译（开启后，停止输入 1.5 秒触发）
  ↓
调用配置的 AI API（传递中文 + 翻译 prompt）
  ↓
API 返回英文翻译结果
  ↓
替换编辑器内容为英文（execCommand / DOM fallback）
  ↓
光标移至末尾，用户可继续编辑或直接发送
  ↓
X 的发送按钮读取内部状态 → 发送翻译后的英文内容
```

```
├── popup.html         # 设置面板界面

└── icon128.png        # 128x128 图标
```

## 7. 技术栈

- **Manifest V3**：Chrome 扩展最新规范
- **Content Script**：注入到 `x.com` / `twitter.com` 页面
- **chrome.storage.sync**：跨设备同步用户配置
- **MutationObserver**：SPA 动态 DOM 变化监听
- **execCommand**：contentEditable 文本操作（兼容 Draft.js）
- **Fetch API**：直接向 AI 提供商发起 HTTP 请求（无后端代理）
